## Why this mission exists

AI-DOS has rules (Missions 001–002) but has never run a full research cycle
under them. This mission validates that AI-DOS can make an evidence-based
product recommendation using only the repository and newly gathered
research. The goal is not to be right; it is to be **transparent,
reproducible, and grounded in real evidence**. This mission recommends; the
operator decides.

## Tasks

1. **Broad scan** — survey candidate benchmark applications against the
   Stage 0 constraints. Record every candidate considered, with a one-line
   discard reason for each non-shortlisted one.
2. **Shortlist exactly three** candidates; the shortlist itself must be an
   evidenced decision.
3. **Deep research** on each of the three: recurring user pain points,
   existing solutions, observable gaps. Problems over ideas.
4. **Evaluate** all three against one rubric (Demonstrates AI-DOS, Portfolio
   value, Revenue potential, Time to MVP, Infrastructure simplicity,
   Operator fit, Future expansion, Implementation risk), with written
   justification for every score.
5. **Failure analysis** for all three, including the recommended one.
6. **Recommend exactly one**, with a Decision Confidence rating
   (High/Medium/Low, no numeric percentages) and a concrete list of what
   would increase confidence.

## Stage 0 constraints (pass/fail gate, not scores)

Buildable by one developer · manageable primarily from an iPhone ·
demonstrates the AI-DOS workflow · clearly bounded MVP · no heavy
infrastructure · compelling portfolio story · realistic revenue potential ·
capable of future iteration.

## Evidence integrity (non-negotiable)

Every citation must come from a source actually accessed during this
mission, with its URL. Nothing cited from memory or plausibility. Claims
that cannot be verified are marked **unverified** and the Decision
Confidence section accounts for them.

## Constraints

- Do not begin implementation.
- Do not create new governance documents; do not modify Principles.md or
  Standards.md unless a genuine error is found (document it if so).
- Update only files necessary to document Mission 003.
- Every commit follows the Knowledge Preservation standard.

## Exit criteria

Broad scan recorded with discard reasons; three candidates deeply researched
with real cited evidence; documented risks for each; exactly one
recommendation with documented reasoning; Decision Confidence complete;
cold-start verification passes by file citation; report complete including
"What AI-DOS Learned" and ending with a single Mission 004 approval
question.
