# Mission 002: Formalize the Knowledge Preservation Standard

**Status:** Complete — see [report.md](report.md)
**Executed by:** Claude Fable 5

## Why this mission exists

AI-DOS itself learned something during Mission 001: a commit that explains
*why* a change was made, and *who benefits later*, functions as
organizational memory in its own right — separate from what the mission
report captures. This mission formalizes that pattern before it is lost.

The operator inserted this mission ahead of the planned research cycle.
"Prove the Loop" (selecting the first benchmark app) is therefore renumbered
from Mission 002 to **Mission 003**; its scope is untouched here.

## Tasks

1. **Principles.md** — add the governing principle "Never make a future
   agent guess" above all others; link to Standards.md with a one-line
   explanation of the Principles/Standards split; record (not build) the
   intent for a future "Repository Intelligence Score."
2. **Create company/Standards.md** — first section: Knowledge Preservation.
   State that memory lives in multiple forms (commits, reports, decisions,
   research) and that commits are the first formalized standard. Document
   the commit format (`Why:` / `Future benefit:` body) and the division of
   memory: reports summarize at the mission level, commits preserve
   rationale at the change level.
3. **Mission 001 addendum** — note in its report that the standard was
   formalized here, based on a pattern observed there. Do not rewrite
   Mission 001's Git history; the organic format is evidence of discovery.
4. **Mission template** — create `/workflow/Templates/MissionTemplate.md`
   (none existed) with a required `Executed by:` field in mission.md and a
   required "What AI-DOS Learned" section in every report. This mission is
   the first executed under the template.
5. **Resequence** — Mission 003 = Prove the Loop, everywhere. No file may
   point to the old numbering unexplained.

## Constraints

- No Mission 003 planning or app selection beyond renumbering.
- Repository Intelligence Score: record intent only, build nothing.
- Do not rewrite history; document discoveries instead of perfecting them.
- Every commit in this mission uses the new commit standard.
- Stop at the exit criteria; further improvements belong to future missions.

## Exit criteria

Principles.md carries the governing principle and Standards link;
Standards.md exists with the Knowledge Preservation standard; Mission 001's
report carries the addendum; the template has both new requirements; no
stale old-numbering references remain; this mission's commits follow the new
standard; this mission's report is complete and ends with the Mission 003
approval question.
