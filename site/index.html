What AI-DOS Is

AI-DOS is a Git-native operating system for AI-assisted software development. Instead of keeping decisions in chat, it records mission briefs, outcomes, evidence, and next steps in versioned files so fresh agents can continue without hidden context.

What AI-DOS Learned

  • M001: Start lean; every file must earn its place.
  • M002: Change-level reasoning belongs in commits; mission-level reasoning belongs in reports.
  • M003: Evidence confidence should route roadmap decisions.
  • M004: Approval is durable only when work is merged into main.
  • M005: A polished visual layer can improve comprehension without replacing canonical records.

Governance Flow

Mission proposed → Mission executed → Mission reviewed → Operator approves → Merged to main → Repository becomes canonical

Evidence-Based Decision Engine

Mission 003 recommended a freelancer invoice follow-up tool with Medium confidence. Because evidence confidence was not High, AI-DOS prioritized validation before build. Roadmap sequencing is evidence-first, not assumption-first.
